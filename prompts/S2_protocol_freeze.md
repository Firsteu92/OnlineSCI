# S2：协议冻结 / Protocol Freeze

## 什么时候使用

正式实验前，需要冻结实验方案时。

## 使用前需要准备

- 已完成 S1（G1 已通过）
- 有明确的研究假设

## 可直接复制的 Prompt

```
# Role
你是一个实验设计专家。请帮我制定和冻结正式的实验协议。

# Research Question
[在此输入研究问题]

# Hypothesis
[在此输入核心研究假设]

# Available Data
[描述可用的数据及其范围]

# Instructions
请帮我制定以下实验协议：

1. **研究假设**：明确可检验的假设（H0/H1）
2. **数据范围**：
   - 训练/测试/验证的划分方式
   - 时间范围
   - 空间范围
   - 排除标准
3. **基线方法**：
   - 比较方法
   - 评价指标（主指标、次指标）
4. **实验设计**：
   - 主实验
   - 消融实验
   - 鲁棒性分析
   - 独立验证
5. **失败标准**：
   - 什么情况下认为假设被证伪
   - 什么情况下实验无效
6. **协议变更规则**：
   - 允许的变更类型
   - 禁止的变更类型
   - 变更必须创建新版本

# Output Format
输出 protocol_v001.md（完整文本）和 protocol_v001.yml（结构化版本）。

# Constraints
- 指标必须可量化
- 失败标准必须明确
- 协议冻结后不得无版本地修改
```

## 期望输出

- `protocol_v001.md` 完整协议
- `protocol_v001.yml` 结构化版本

## 人类审查重点

- [ ] 指标是否合理
- [ ] 失败标准是否明确
- [ ] 是否有未考虑的偏差

## 保存位置

`projects/pXX/protocols/protocol_v001.md`、`projects/pXX/protocols/protocol_v001.yml`
