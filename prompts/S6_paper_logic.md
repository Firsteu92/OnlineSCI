# S6：论文逻辑 / Paper Logic

## 什么时候使用

已有充分的实验证据，需要组织论文结构和叙事线时。

## 使用前需要准备

- G3、G4 均已通过
- 有完整的实验证据和证据账本

## 可直接复制的 Prompt

```
# Role
你是一个论文结构设计专家。请帮我建立论文的主线和详细提纲。

# Research Question
[在此输入研究问题]

# Key Findings
[列出核心发现]

# Evidence Summary
[粘贴或引用证据账本中的关键结论]

# Target Journal
[输入目标期刊名称]

# Instructions
请完成以下任务：

1. **论文主线**：用 3-5 句话概括论文的核心故事
2. **核心贡献**：列出 2-4 条清晰、可验证的贡献声明
3. **章节提纲**：每个章节的段落级提纲，标注每段要传达的要点
4. **证据分配**：为每个主要论点分配：
   - 支撑它的实验证据（运行 ID）
   - 支撑它的参考文献
   - 分配的图件编号
5. **图件规划**：建议核心图件的数量和布局

# Output Format
输出：
1. paper_story.md — 主线、贡献和章节结构
2. detailed_outline.md — 段落级详细提纲
3. contribution_plan.md — 贡献安排
4. section_evidence_map.csv — 章节-证据映射

# Constraints
- 不要编造不在证据账本中的结论
- 每个论点必须有对应的证据
- 不要过度承诺贡献
- 不写入完整正文（这是下一步的工作）
```

## 期望输出

- 论文主线文档
- 详细提纲
- 证据分配矩阵

## 人类审查重点

- [ ] 故事线是否清晰、有说服力
- [ ] 每个论点是否有足够证据

## 保存位置

`projects/pXX/manuscript/planning/`

## 下一门槛
