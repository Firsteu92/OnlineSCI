# S1：创新判断 / Idea Evaluation

## 什么时候使用

文献检索完成后，需要判断研究想法是否可行、是否有创新性时。

## 使用前需要准备

- 已完成文献检索（S1_deep_research）
- 有核验过的文献列表

## 可直接复制的 Prompt

```
# Role
你是一个科研选题评审专家。请基于已核验的文献，评价我的研究想法。

# Research Question
[在此输入研究问题]

# Literature Summary
[粘贴文献检索的结果，特别是最接近已有工作和研究空白]

# Instructions
请从以下维度评价该研究想法：

1. **新颖性**：该想法与已有工作相比的新意在哪里？（0-10 分）
2. **科学意义**：如果成功，对领域的贡献有多大？（0-10 分）
3. **可验证性**：该假设是否可以通过实验/数据检验？（0-10 分）
4. **可行性**：在可用资源下是否可行？（0-10 分）
5. **发表潜力**：适合什么级别的期刊？

# Fatal Flaw Check
检查是否存在以下致命缺陷：
- 核心假设已被证伪
- 所需数据不可获取
- 方法路径不可行
- 时间/资源需求远超可用

# Decision
给出最终决定之一：
- **PROCEED**：可以继续
- **PROCEED_WITH_CHANGES**：需要修改以下内容后再继续...
- **HOLD**：需要补充以下信息后再判断...
- **STOP**：不建议继续，理由...

# Constraints
- 所有判断必须基于已核验的真实文献
- 明确指出哪些是事实、哪些是判断
- 如果证据不足，标注"待补充"

# Output
输出完整的评估报告。
```

## 期望输出

- 多维度评分
- 致命缺陷检查
- 最终决定（PROCEED / HOLD / STOP）

## 人类审查重点

- [ ] AI 的判断是否合理
- [ ] 是否有 AI 忽略的关键因素
