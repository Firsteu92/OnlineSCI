# Review Checklist — 审查清单

每个项目在各阶段使用本清单进行审查。审查者在 PR 中逐项打勾。

---

## S1 文献审查

### 文献完整性
- [ ] 文献检索覆盖核心方向
- [ ] 每篇文献的 DOI 已核验
- [ ] 无编造引用
- [ ] 关键文献未被遗漏

### 创新判断
- [ ] 研究空白已识别
- [ ] 创新性已评估
- [ ] 致命缺陷已检查

---

## S2 协议审查

### 协议完整性
- [ ] 假设明确可检验
- [ ] 数据范围明确
- [ ] 指标可量化
- [ ] 失败标准明确
- [ ] 协议版本已标注

---

## S4 实验审查

### 运行记录
- [ ] 运行 Manifest 完整
- [ ] 参数和环境已记录
- [ ] 失败实验已记录

### 协议符合性
- [ ] 实验遵循冻结协议
- [ ] 所有偏差已记录
- [ ] 泄漏检查已执行

### 可复现性
- [ ] 核心结果可复现
- [ ] 环境已记录

---

## S5 证据审查

### 证据完整性
- [ ] 声明-证据-图表映射完整
- [ ] 无证据声明已标注
- [ ] 冲突结果已记录
- [ ] 结论强度匹配证据强度

---

## S7 论文审查

### 科学内容
- [ ] 研究问题清晰、重要
- [ ] 方法适合问题
- [ ] 结论有数据支撑

### 写作质量
- [ ] 结构合理、逻辑连贯
- [ ] 语言流畅
- [ ] 无 AI 典型套话

### 图表
- [ ] 清晰完整
- [ ] 与正文一致

### 写作权限
- [ ] 在 G3/G4 允许的范围内写作

---

## S8 投稿前审查

### 综合检查
- [ ] 科学内容正确
- [ ] 证据完整
- [ ] 格式符合目标期刊
- [ ] 投稿材料齐全
- [ ] AI 使用声明已撰写
- [ ] 数据可用性声明已撰写

---

## 使用方法

1. 审查者在 PR 评论中逐项检查
2. 所有必选项通过后，Approve PR
3. 如有重大问题，Request Changes 并具体说明
